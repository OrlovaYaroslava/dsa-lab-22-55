# Чек-лист для тестирования функции get_triangle_type

## Позитивное тестирование (проверка корректных данных):
1. Треугольник равносторонний:
   - Входные данные: 3, 3, 3
   - Ожидаемый результат: "equilateral"

2. Треугольник равнобедренный:
   - Входные данные: 5, 5, 3
   - Ожидаемый результат: "isosceles"

3. Треугольник разносторонний:
   - Входные данные: 3, 4, 5
   - Ожидаемый результат: "nonequilateral"

## Негативное тестирование (проверка некорректных данных):
1. Одна из сторон отрицательная:
   - Входные данные: -3, 4, 5
   - Ожидаемый результат: Исключение IncorrectTriangleSides (Стороны треугольника должны быть положительными числами.)

2. Одна из сторон равна нулю:
   - Входные данные: 0, 4, 5
   - Ожидаемый результат: Исключение IncorrectTriangleSides (Стороны треугольника должны быть положительными числами.)

3. Стороны не могут образовать треугольник (нарушена теорема о треугольниках):
   - Входные данные: 1, 1, 3
   - Ожидаемый результат: Исключение IncorrectTriangleSides (Сумма любых двух сторон треугольника должна быть больше третьей.)

4. Одна из сторон больше суммы двух других:
   - Входные данные: 10, 3, 2
   - Ожидаемый результат: Исключение IncorrectTriangleSides (Сумма любых двух сторон треугольника должна быть больше третьей.)

5. Все стороны отрицательные:
   - Входные данные: -5, -5, -5
   - Ожидаемый результат: Исключение IncorrectTriangleSides (Стороны треугольника должны быть положительными числами.)
